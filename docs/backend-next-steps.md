# 后端后续工作清单

本文档只记录后端需要继续完善的部分。前端页面后续会重写，因此这里不包含 UI、视觉、组件和页面交互计划。

## 当前后端基线

当前后端已经具备 MVP 能力：

| 模块 | 当前状态 |
| --- | --- |
| 认证 | 邮箱密码注册、登录、退出、当前用户，会话使用 HttpOnly Cookie |
| 数据库 | PostgreSQL + Prisma，核心业务表已迁移 |
| 情侣空间 | 创建、邀请码加入、详情、更新、删除 |
| 关系数据 | 纪念日、日记、心情、愿望、照片记录、时间胶囊 CRUD |
| Agent | DeepSeek provider 已接入关系助手，支持 fallback 和调用日志 |
| 测试 | 已有 API E2E 脚本和 DeepSeek smoke 脚本 |

## P0：后端必须补强

| 工作项 | 目标 | 验收标准 |
| --- | --- | --- |
| 认证安全增强 | 让登录体系达到可上线的基本安全水平 | 注册/登录加入限流；密码策略更清晰；失败登录不泄露账号状态 |
| 密码找回 | 用户忘记密码时可以安全恢复 | 支持重置 token、过期时间、一次性使用、邮件发送接口预留 |
| 邮箱验证 | 避免无效邮箱注册 | 注册后生成验证 token；用户可重新发送验证邮件；未验证状态可限制敏感操作 |
| 会话管理 | 用户可查看和撤销登录会话 | `AuthSession` 支持设备信息、IP、UA；支持撤销其他会话 |
| 权限策略统一 | 降低业务接口权限遗漏风险 | 抽象统一的 couple member/owner/author 校验工具；补齐测试 |
| API 错误规范 | 让前端稳定处理错误 | 所有 API 使用统一错误结构、错误码、HTTP status |
| 输入校验补齐 | 防止脏数据进入数据库 | 所有创建/更新接口都有 zod schema；日期、金额、枚举、长度边界测试覆盖 |
| 接口 E2E 扩展 | 避免后端回归 | 覆盖纪念日、心情、愿望、照片、时间胶囊、Agent 日志 |

## P1：核心业务完善

| 工作项 | 目标 | 验收标准 |
| --- | --- | --- |
| 邀请机制升级 | 邀请码更安全可控 | 邀请码可刷新、可过期、可禁用；加入空间需要双方确认 |
| 情侣解绑流程 | 避免误删和数据纠纷 | 支持解绑申请、确认、冷静期、数据保留策略 |

| 时间胶囊安全 | 保证未到期不可读 | 数据库层或服务层保证未解锁内容不会返回；开启操作有审计记录 |
| Agent 记忆管理 | 用户可控地使用 AI 记忆 | 可查看、创建、更新、删除 `AgentMemory`；可关闭记忆 |

## 推荐开发顺序

1. 先补齐认证安全：邮箱验证、密码找回、登录限流、会话管理。
2. 再补业务权限测试：所有资源 CRUD 都覆盖成员、作者、创建者边界。
3. 然后做 Agent 工具调用：先生成草稿，用户确认后写库。
4. 接着做图片上传和对象存储。
5. 最后做通知、定时任务、审计日志和生产部署。
