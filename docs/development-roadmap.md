# 开发计划表

## 阶段一：基础框架与产品骨架

| 任务 | 交付物 | 验收标准 | 优先级 |
| --- | --- | --- | --- |
| 初始化 Next.js 全栈项目 | App Router、TypeScript、Tailwind | `npm run dev` 可启动 | P0 |
| 建立数据库模型 | Prisma schema | `npm run db:generate` 成功 | P0 |
| 建立 AI Agent API | `/api/agents/relationship` | 无 API Key 时有本地 fallback | P0 |
| 建立首页工作台 | Dashboard UI | 首屏展示心情、纪念日、愿望、Agent | P0 |
| 建立文档 | 需求分析、路线图 | 文档可指导后续开发 | P0 |

## 阶段二：MVP 可用闭环

| 任务 | 交付物 | 验收标准 | 优先级 |
| --- | --- | --- | --- |
| 用户登录 | 邮箱/第三方登录 | 用户可注册登录退出 | P0 |
| 情侣绑定 | 邀请码与邀请链接 | 两个账号可进入同一空间 | P0 |
| 纪念日 CRUD | 增删改查与提醒字段 | 首页显示最近纪念日 | P0 |
| 情侣日记 CRUD | 草稿、可见范围、共同日记 | 可按权限查看 | P0 |
| 心情打卡 | 心情、能量、照顾偏好 | 首页展示双方最新状态 | P0 |
| 愿望清单 CRUD | 分类、状态、进度 | 可按状态筛选 | P0 |

## 阶段三：AI 深度集成

| 任务 | 交付物 | 验收标准 | 优先级 |
| --- | --- | --- | --- |
| 意图识别 | Agent 自动识别日记/纪念日/愿望 | 用户一句话生成草稿 | P1 |
| 工具调用 | createAnniversary、createWish 等工具 | 写入前必须展示确认卡片 | P1 |
| Agent 记忆 | 可查看、可删除的偏好记忆 | 用户能关闭和清除记忆 | P1 |
| 冷静沟通区 | 分步表达模板 | 输出可保存为复盘记录 | P1 |
| 约会生成器 | 城市、预算、时间偏好 | 生成 1 个主方案和 2 个备选 | P1 |

## 阶段四：长期留存与体验完善

| 任务 | 交付物 | 验收标准 | 优先级 |
| --- | --- | --- | --- |
| 共同相册 | 上传、归档、事件关联 | 可批量上传与查看 | P1 |
| 时间胶囊 | 创建、锁定、定时打开 | 未到期不可打开 | P1 |
| 周报/月报 | 自动关系回顾 | 用户确认后保存 | P2 |
| 通知系统 | 纪念日、任务、心情提醒 | 用户可配置频率 | P2 |
| 移动端优化 | 响应式与触摸体验 | 手机端不溢出、不遮挡 | P1 |

## 阶段五：上线准备

| 任务 | 交付物 | 验收标准 | 优先级 |
| --- | --- | --- | --- |
| 测试覆盖 | 单元测试、API 测试、关键 E2E | 核心流程可自动验证 | P0 |

| 监控日志 | 错误日志、Agent 调用日志 | 可定位线上问题 | P1 |
| 数据备份 | 数据库备份策略 | 可恢复关键数据 | P1 |
