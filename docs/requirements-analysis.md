# 情侣私密生活协作系统需求分析

## 产品定位

本系统面向情侣双方，提供一个私密、低操作成本、长期可沉淀的共同空间。核心价值不是社交展示，而是帮助两个人记录关系、规划生活、表达状态、降低沟通成本，并用 AI Agent 辅助完成高频小任务。

## 角色与边界

| 角色 | 目标 | 权限边界 |
| --- | --- | --- |
| 未登录用户 | 了解产品并进入注册 | 不能访问情侣空间数据 |
| 普通用户 | 创建或加入情侣空间 | 可管理个人资料、个人日记、自己的心情 |
| 情侣成员 | 共同管理两人空间 | 可访问双方确认共享的数据 |
| 空间创建者 | 邀请另一半、管理空间基础设置 | 不能绕过对方隐私权限 |
| AI Agent | 辅助整理、提醒、生成建议 | 不能自动写入敏感数据，必须等待确认 |

## 功能需求表

| 模块 | 核心功能 | 完整能力 | 无脑化设计 | 优先级 |
| --- | --- | --- | --- | --- |
| 用户与登录 | 邮箱/手机号/第三方登录 | 资料、头像、偏好、注销 | 一键进入，首次自动引导 | P0 |
| 情侣绑定 | 邀请码/邀请链接 | 绑定确认、解绑申请、空间名称 | 一方生成链接，另一方点确认 | P0 |
| 情侣首页 | 恋爱天数、纪念日、心情、愿望 | 今日待办、最近动态、AI 建议 | 首页只展示今天最重要的事 | P0 |
| 纪念日 | 新增、编辑、提醒 | 倒计时、重复规则、庆祝建议 | 输入一句话自动识别日期 | P0 |
| 情侣日记 | 个人/共同日记 | 图片、标签、可见范围、草稿 | 默认模板，AI 自动润色标题 | P0 |
| 心情打卡 | 心情、能量、压力、照顾方式 | 趋势统计、关心提醒 | 表情/选项优先，输入可选 | P0 |
| 愿望清单 | 旅行、电影、餐厅、目标 | 预算、时间、状态、双方确认 | 一句话添加，AI 自动分类 | P0 |
| 相册 | 上传、归档、事件关联 | 批量上传、年度回忆、封面 | 上传后自动聚类与命名 | P1 |
| 时间胶囊 | 未来信、约定、定时开启 | 锁定期、双方留言、打开记录 | 默认 30/100/365 天 | P1 |
| 冷静沟通区 | 情绪表达与复盘 | 感受、需要、请求、后续约定 | AI 分步提问，每步只问一个点 | P1 |
| 小任务 | 关心提醒、约会任务 | 周期任务、完成反馈、轻提醒 | 一键生成本周小任务 | P1 |
| 关系报告 | 周报、月报、年度回忆 | 情绪趋势、完成愿望、精选照片 | 系统自动生成，用户只确认 | P2 |
| 隐私与安全 | 可见范围、删除、导出 | 操作日志、敏感操作二次确认 | 默认保护个人边界 | P0 |

## AI Agent 需求

| Agent | 负责事项 | 输入 | 输出 | 写入规则 |
| --- | --- | --- | --- | --- |
| 关系助手 Agent | 总入口、意图识别 | 用户自然语言 | 简短建议与快捷动作 | 只建议，不直接写入 |
| 纪念日 Agent | 日期识别、提醒建议 | “我们 8 月 3 日在一起” | 纪念日草稿 | 用户确认后创建 |
| 日记 Agent | 陪写、润色、标题生成 | 碎片描述 | 日记草稿 | 用户确认可见范围 |
| 愿望 Agent | 清单分类、计划拆解 | “想去日本看樱花” | 愿望卡片草稿 | 双方共享前确认 |
| 约会 Agent | 约会方案 | 城市、预算、时间、偏好 | 1 个推荐方案与备选 | 可加入任务或愿望 |
| 沟通 Agent | 冲突修复 | 情绪描述 | 感受-需要-请求模板 | 不判断谁对谁错 |
| 回忆 Agent | 周报/月报 | 系统内记录 | 回顾文案 | 用户确认后保存 |

## 非功能需求

| 类型 | 要求 |
| --- | --- |
| 易用性 | 高频功能 3 次点击内完成，表单采用渐进式输入 |
| 隐私 | 私密、对方可见、共同可见三层权限 |
| 性能 | 首页首屏 2 秒内可交互，图片走对象存储与 CDN |
| 安全 | 敏感操作二次确认，AI 不输出控制或监控建议 |
| 可维护性 | 前后端统一 TypeScript，数据库模型清晰可迁移 |
| 可扩展性 | Agent 能力按工具模块拆分，后续可接多 Agent 编排 |

## 数据模型概览

当前初版已建立以下核心模型：`User`、`Couple`、`CoupleMember`、`Anniversary`、`DiaryEntry`、`MoodCheckIn`、`Wish`、`Photo`、`TimeCapsule`、`AgentMemory`、`AgentRun`。
